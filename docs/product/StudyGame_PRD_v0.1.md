# StudyGame PRD v0.1

> 项目：StudyGame  
> 产品形态：半开放养成 RPG 儿童英语生活冒险游戏  
> 上位依据：[教学玩法重构策划_v0.1.md](/home/xionglei/GameProject/SutdyGame-godot/docs/product/教学玩法重构策划_v0.1.md:1)  
> 美术方向：原创儿童向生活冒险插画风  
> 目标用户：7-12 岁儿童，首发重点覆盖小学四年级左右玩家  
> 引擎：Godot 4.6.x  
> 日期：2026-06-01

## 1. 产品概述

StudyGame 不再定义为“带地图和任务的校园英语学习应用”，而是一款儿童生活冒险游戏。玩家从 `home` 出发，在 `school`、`town`、`transport` 和 `world` 逐步扩展的空间中探索、照顾宠物、完成委托、购买物品、换装、收集奖励，并在这些行为中自然使用英语。

一句话定位：

`StudyGame` 是一款以 `home -> school -> town -> transport -> world` 为探索路径、以 `记忆宫殿` 为底层学习结构、以 `半开放养成 RPG` 为外层体验的儿童英语生活冒险游戏。

产品的关键判断有三条：

1. 孩子前台不应该先感知到“上课”和“做题”。
2. 英语内容必须嵌入宠物、商店、家园、旅行、节日和同伴事件中。
3. A-Z 记忆宫殿、地图结构和教材主线必须属于同一套产品系统。

## 2. 产品目标

### 2.1 儿童目标

- 愿意主动进入游戏，而不是被动进入学习流程。
- 在探索、养宠、购物、换装和剧情推进中自然接触英语。
- 对旧知识点产生“我见过、我用过、我知道它在地图哪里”的熟悉感。
- 感受到长期成长，而不是只完成一次性关卡。

### 2.2 家长目标

- 能看见孩子学过哪些词、句型和场景表达。
- 能理解游戏不是单纯娱乐，也不是直白课堂壳。
- 能通过家长层确认阶段性奖励，而不是被迫长期高频介入。
- 能判断内容适龄、安全、非沉迷、与教材难度有连接。

### 2.3 项目目标

- 建立可扩展的 `序章 + 七篇` 教学玩法主线。
- 验证 `记忆宫殿 + 半开放 RPG + 教材递进` 的统一可玩性。
- 保留现有世界地图、家长摘要、点击/拖拽交互和任务系统资产。
- 建立供故事线、开发拆解和任务配置持续维护的产品真相。

## 3. 目标用户

### 3.1 核心玩家

- 年龄：7-12 岁
- 首发重点：9-10 岁，四年级左右玩家
- 基础差异：允许从“三年级下册之前基础”补起，兼容基础较弱用户
- 偏好：宠物、换装、收集、朋友、家园、小镇、节日、轻剧情
- 反感：考试感、长文字、失败惩罚、数值压迫、复杂系统说明

### 3.2 家长用户

- 关注学习效果、内容安全、难度匹配、孩子是否愿意持续回访
- 希望得到简洁可解释的学习摘要
- 允许阶段性奖励，但不希望游戏内经济完全依赖家长投喂

## 4. 产品定位

### 4.1 外层定位

外层是半开放养成 RPG。

孩子感知到的产品关键词固定为：

- home
- pet
- school
- town
- shop
- dress up
- help
- trip
- show
- wish

不前台直出的学习化词汇：

- 课时
- Unit
- L1 / L2 / L3
- 单词表
- 句型表
- 复习题
- Lesson
- Test

### 4.2 体验关键词

- 半开放探索
- home to school to town to transport to world
- 记忆宫殿地图
- 宠物陪伴
- 换装收集
- 商店消费
- 节日与旅行剧情
- 家长可解释学习成果

## 5. 三条主线

### 5.1 记忆宫殿线

A-Z 记忆宫殿是产品底层学习骨架，不是彩蛋系统。

硬规则：

1. `一字母一主编码`
2. `一主编码一固定位置`
3. `A-Z 主编码表冻结`
4. `新增场景只能扩展场景词和任务词，不得替换主锚点`
5. `home -> school -> town -> transport -> world` 是长期固定路线

功能定位：

- 字母识别入口
- 单词图像入口
- 地点回忆入口
- 后续任务与复习回调入口

开放方式固定为：

- 序章完成后，全量 A-Z 锚点正式开放
- 在此之前只暴露序章教学所需的强锚点或基础锚点
- 当前实现边界：全量开放指锚点可点击、可进入首访记忆对话；`Memory Spark` 回访提取仍是 pilot 范围，先覆盖少量锚点，再按验证结果扩展到全量 A-Z。

### 5.2 半开放 RPG 线

产品采用 `一条主线 + 少量支线 + 稳定回访空间`。

玩家不是在刷教材，而是在：

- 接委托
- 探索地图
- 照顾宠物
- 购物
- 换装
- 布置家园
- 推进剧情

英语内容只作为解决问题和推进剧情的工具存在。

### 5.3 教材递进线

采用 `序章 + 七篇`：

- `序章`：补三年级下册之前的基础
- `第 1-7 篇`：继续沿用现有七篇教材主线包装

序章是独立基础层，不是“第 0 篇教材”。

序章固定包含：

- A-Z
- 字母名
- 基础发音感知
- 最核心生活名词
- 最基础动作词
- 最基础指示表达

七篇继续承载三年级下册到六年级下册内容，但前台继续用剧情和任务包装，不直接暴露教材结构。

## 6. 世界观与角色

### 6.1 世界观

游戏发生在原创的“星光小镇 / Sunshine Town”。小镇中心有一座故事钟塔，语言星片散落在 `home`、`school`、`town` 商业街、自然花园、交通枢纽和世界边框区域。玩家作为新搬来的孩子，被邀请成为“星路修复员”，通过帮助朋友、照顾宠物、准备节日、处理生活委托和完成旅行分享，让小镇逐步恢复光彩。

### 6.2 主要角色

| 角色 | 定位 | 功能 |
|---|---|---|
| 玩家主角 | 新搬到星光小镇的孩子 | 探索、成长、装扮、剧情推进主体 |
| Mina | 活泼同伴 | home 到 school 的起步引导，社交、节日、换装事件 |
| Leo | 工具和地图爱好者 | 路线、计划、调查、时间与制作事件 |
| Nora | 阅读和表演爱好者 | 故事、书店、剧场、阅读与展示事件 |
| Coco | 宠物伙伴 | 陪伴、提示、低压互动、序章和前期核心载体 |
| Grandma Joy | 家庭与生活长辈 | 健康、节日、家庭、照顾类事件 |
| Keeper Ray | 故事钟塔看守人 | 主线章节解锁、世界地图推进、阶段奖励解释 |

## 7. 核心玩法

### 7.1 操控方式

- 2D 俯视或 2.5D 视角
- WASD / 方向键移动
- 鼠标点击 UI、选项、物件和地图热点
- 靠近 NPC 或对象时显示互动提示
- 场景通过门、路口、地图热点切换
- 世界地图支持移动和浏览，前期从 `home + school` 起步

### 7.2 单次核心循环

1. 在 `home` 或小镇接到一个目标
2. 探索地图、对话、观察、购物或照顾宠物
3. 用英语理解提示并完成交互
4. 获得 `Coins / 道具 / 外观 / 宠物材料`；词汇/表达记录只进入家长层和报告层
5. 回家养成、换装、布置、开启下一步剧情

### 7.3 长线循环

1. 剧情推进
2. 宠物成长
3. 皮肤/装扮收集
4. 家园和地图解锁
5. 教材难度递进
6. 旧知识点回访

### 7.4 英语学习原则

- 每次任务只突出少量核心语言点
- 英语必须绑定动作、地点、选择、购买、照顾或路线
- 失败只给提示和重试，不切断主奖励
- 中文承担理解支撑，英语承担核心表达和识别
- 同一知识点通过地图、商店、宠物、节日和旅行持续复现

## 8. 章节结构

### 8.1 总结构

产品主线固定为：

1. `序章：Letters, Home, My First Pet`
2. `第一篇：小镇新朋友`
3. `第二篇：校园发现日`
4. `第三篇：节日校园周`
5. `第四篇：城市委托簿`
6. `第五篇：计划家与小记者`
7. `第六篇：世界小小行动队`
8. `第七篇：毕业旅行星图`

### 8.2 章节职责

| 章节 | 剧情目标 | 语言目标 | 地图主场景 | 核心玩法形态 |
|---|---|---|---|---|
| 序章 | 熟悉 home、宠物和基础锚点 | A-Z、基础名词、基础动作、指示表达 | home、school 首屏周边 | 养宠、找物、指认、基础探索 |
| 第一篇 | 认识小镇伙伴，进入第一圈生活区域 | 问候、时间、天气、基础生活表达 | school、garden、town 起步节点 | 认识伙伴、小活动、第一次协作 |
| 第二篇 | 让学校成为真实生活空间 | 学校词、作息、食物、动物、能力、规则 | school、home 回访点 | 带路、认路、陪伴探索、整理、照顾、规则理解 |
| 第三篇 | 让学校进入节日与动态事件状态 | 方位、课程、星期、进行时 | school、park、节日节点 | 准备活动、观察事件、现场判断 |
| 第四篇 | 开始独立处理 town 生活委托 | 购物、点餐、借物、宠物、情绪、节日生活 | town 商店群 | 购买、照顾、帮助、选择 |
| 第五篇 | 从帮忙者升级成计划者和记录者 | 计划、研究、问路、月份、过去时 | town、transport、资料节点 | 制作、采访、路线、调查 |
| 第六篇 | 从个人生活走向公共行动 | 将来时、建议、比较、健康、环保 | transport、world 延展区 | 准备、协作、展示、计划实施 |
| 第七篇 | 整合小学阶段表达完成毕业旅行分享 | 综合表达、比较、建议、旅行介绍 | world 边框区、全图回访 | 旅行准备、整合展示、最终回访 |

## 9. 系统设计

### 9.1 任务与前台表达

前台统一使用偏游戏化表达：

- Quest
- Wish
- Help
- Trip
- Show
- Shop
- Pet Care

系统内部允许保留配置化任务结构，但显示层不直接暴露 `Lesson / Review / Test / Unit`。

当前前台显示已按 `Quest Diary` 方向收口，内部仍保留 `QuestDiary` 作为稳定技术名。

### 9.2 宠物系统

宠物系统首版固定为轻养成，不做战斗化。

状态：

- 饥饿
- 清洁
- 心情
- 亲密度

互动：

- 喂食
- 清洁
- 陪玩
- 睡觉

职责：

1. 长期回访动机
2. 低压学习触发器
3. 序章和前期章节的重要场景载体

### 9.3 装扮与外观系统

装扮系统是长期目标，不是一次性奖励。

承载内容：

- 角色服装
- 配饰
- 包
- 房间角落装饰
- 宠物外观或用品

作用：

1. 提供持续目标
2. 稀释做题感
3. 让奖励与自我表达绑定

### 9.4 商店系统

商店固定拆成三类：

- `Pet Shop`
- `Clothes Shop`
- `Home Decor / General Store`

用途：

- 消耗 `Coins`
- 购买宠物耗材
- 购买外观与装饰
- 让词汇在真实消费场景中复现

### 9.5 经济系统

双层货币固定为：

- `Coins`
- `Parent Bonus`

#### Coins

来源：

- 完成事件
- 探索
- 照顾宠物
- 剧情推进
- 常规活动

用途：

- 宠物耗材
- 普通装扮
- 普通装饰

#### Parent Bonus

来源：

- 家长确认后追加

用途：

- 稀有奖励
- 限定外观
- 阶段性激励

边界：

- 不允许把整套经济系统做成“刷题换金币”
- 不允许用一种货币混合全部成长与家长奖励

### 9.6 家长层

孩子层看到：

- 目标 / 生活事件 / 冒险日志
- 地图
- 宠物
- 商店
- 奖励

家长层看到：

- 学过哪些词和句型
- 当前剧情进度
- 哪些知识点需要回访
- 是否发放 `Parent Bonus`

## 10. MVP 与版本规划

### 10.1 当前 MVP 方向

现有可玩切片不再前台表述为“校园三任务链”，而是一个生活事件型垂直切片。`MVP 0.2` 在本文中视为历史验证切片，用于证明起步路线、任务包装和家长摘要链路成立，不再作为完整产品边界。

该切片的产品解释是：

- 新存档默认从 `HomeLayer` 进入；打开 `world_overview` 后看到 `home + school` 起步构图
- 通过 Mina 进入 school 起步事件
- 通过 Leo 完成空间整理事件
- 通过 Nora 完成 garden 观察事件
- 通过奖励和家长摘要证明“生活冒险中发生了学习”

### 10.2 Prototype 0.1（历史原型验证）

目标：验证移动、对话、地图热点和第一段游戏化目标是否可理解。

范围：

- 可移动主角
- 一张小镇测试地图
- 1 个起步伙伴
- 1 个生活事件任务
- 基础奖励反馈

### 10.3 MVP 0.2（历史验证切片）

目标：形成成人熟练试玩 2-5 分钟闭环，并验证 home -> school -> quest -> reward -> parent summary 这条最小体验链。

范围：

- `HomeLayer` 默认起步，打开 `world_overview` 后看到 `home + school` 同屏起步
- home / school / classroom / garden 基础链路
- 1 个 home-first 微序章事件 `Welcome Box`，再进入 4 个正式 MVP 前台事件：`First Trip`、`Walk With Mina`、`Room Helper`、`Bird Watch`
- 基础奖励反馈
- 简单存档
- 家长摘要基线

### 10.4 Alpha 0.3

目标：建立序章与前两篇可持续循环。

范围：

- 序章正式进入
- 宠物基础养成
- 商店基线
- 换装与家园角落基线
- 8-10 个剧情任务

### 10.5 Beta 0.4

目标：形成 `序章 + 七篇` 的稳定内容骨架。

范围：

- 完整章节结构
- 商店与养成闭环
- 地图扩展和回访机制
- 家长奖励确认机制增强
- 儿童测试与难度调优

## 11. 美术与体验要求

### 11.1 美术方向

- 原创儿童向生活冒险插画风
- 明亮、精致、梦幻、适龄
- 不做单一粉色界面
- 场景应像玩具小镇、童话校园、家庭角落和旅行展示区的结合

### 11.2 体验要求

- 文本短
- 目标明确
- 反馈温和
- 图像强于抽象说明
- 宠物和场景变化承担情绪反馈
- UI 不出现明显课堂术语

## 12. 工程与数据结构影响

### 12.1 保留的方向

- 世界地图主结构
- `home -> school -> town -> transport -> world` 方向
- 对话系统
- 点击/拖拽/轻交互系统
- 家长摘要
- A-Z 锚点和世界地图热点体系

### 12.2 旧口径与当前表达

- `QuestDiary`：仅作为内部场景/脚本稳定名；儿童前台统一显示 `Quest Diary` 冒险日志。
- `School Tour`：旧教学口径，仅允许出现在兼容说明或迁移记录中；儿童前台统一重述为 `Walk With Mina` 一类生活事件。
- `ReviewChallenge`：旧内部/报告兼容口径；儿童前台统一包装成 `Story Show`、展演、旅行准备或宠物/城镇事件。
- 现有 MVP 三关：在产品叙事里重述为生活事件任务，不再称作课堂关卡或 L1/L2/L3 链。

### 12.3 数据结构方向

后续数据结构要兼容：

- 序章基础层
- 七篇章节层
- 地图锚点层
- Quest Diary 显示层
- 宠物状态
- 经济系统
- 家长层学习摘要

## 13. 成功指标

### 13.1 产品指标

- 新用户 5 分钟内理解移动、对话、接目标和完成一次游戏化事件
- 孩子完整试玩后愿意继续探索、养宠或换装
- 地图、锚点和剧情推进不会被感知为“课堂流程”

### 13.2 学习指标

- 孩子能通过地点回忆出部分字母或单词
- 同一知识点在多个生活场景中被自然复现
- 家长能说出孩子学过的词、句型或生活表达

### 13.3 系统指标

- 目标 / 生活事件 / 冒险日志完成率
- 地图回访率
- 宠物互动频次
- 装扮与商店使用率
- 家长层打开率与奖励确认率

## 14. 风险与对策

| 风险 | 影响 | 对策 |
|---|---|---|
| 产品重新滑回课堂壳 | 孩子第一印象变差 | 前台禁用课时化表达，任务统一生活事件包装 |
| A-Z 锚点频繁改写 | 记忆宫殿失效 | 主编码表冻结，新增场景不得替换主锚点 |
| 经济系统变成刷题换币 | 学习动机异化 | 采用 `Coins + Parent Bonus` 双层货币 |
| 宠物和商店喧宾夺主 | 主线推进失焦 | 用章节节奏控制开放层级，先主线后支线 |
