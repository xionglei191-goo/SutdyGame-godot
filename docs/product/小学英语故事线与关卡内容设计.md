# 小学英语故事线与生活事件/委托内容设计

> 版本：v0.2  
> 日期：2026-06-01  
> 上位依据：`docs/product/教学玩法重构策划_v0.1.md`  
> 适用范围：序章 + 七篇主线、生活事件配置、地图扩展、家长说明  
> 内容来源：`curriculum/小学英语重点分析` 中三年级下册至六年级下册内容；序章为基础层原创内容  
> 文档职责：作为 `故事包装 / 章节推进 / 生活事件命名 / 场景扩展 / 回访设计` 的统一真相来源

## 1. 文档定位

本文件不再把主线内容理解为“7 册教材拆成 75 关”的课程清单，而是把这些内容重新组织成一条儿童生活冒险主线。

这份文档用于统一以下事情：

1. `序章 + 七篇` 的故事推进真相。
2. 每篇的 `剧情目标 / 语言目标 / 地图主场景 / 核心玩法形态`。
3. 每个教材 Unit 在儿童前台应被包装成什么生活事件。
4. 后续生活事件配置、地图扩展、奖励投放、回访任务应遵守的内容边界。
5. 家长层解释口径与儿童前台体验的分离方式。

儿童前台看到的是：

- help
- pet care
- shop
- dress up
- trip
- show
- wish

儿童前台不直接看到的是：

- lesson
- unit
- vocabulary
- pattern
- review
- test

## 2. 新故事真相与硬约束

### 2.1 产品真相

`StudyGame` 的前台真相不是课程化前台，而是一款以 `home -> school -> town -> transport -> world` 为探索路径的儿童生活冒险游戏。

英语在系统里的职责是：

`解决生活问题、完成委托、照顾宠物、理解世界并推进剧情`

而不是直接以前台课程目标出现。

### 2.2 世界路径真相

地图与章节推进必须共享同一条空间路径：

`home -> school -> town -> transport -> world`

含义固定如下：

- `home`：安全区、起步区、宠物和家园区
- `school`：熟悉区、伙伴区、日常规则区
- `town`：委托区、商店区、消费区、社交区
- `transport`：路线区、问路区、出行准备区
- `world`：旅行区、展示区、公共行动区

### 2.3 A-Z 锚点真相

记忆宫殿与故事线是同一个系统，必须遵守：

1. `A-Z 主编码表冻结`
2. `一字母一主编码，一主编码一固定位置`
3. 序章内只暴露序章必需锚点和少量强锚点
4. 序章完成后，全量 A-Z 锚点进入正式交互状态
5. 后续新增场景只能扩展场景词、任务词和事件，不得改写主锚点

实现边界：全量 A-Z 锚点进入正式交互状态，指地图锚点可点击并能承载首访记忆对话；`Memory Spark` 回访提取仍是 pilot 机制，不在首版默认承诺 26 个锚点全部拥有同等回访卡。

### 2.4 生活事件/委托真相

原有教材单元或旧关卡映射必须重述为生活事件/委托，而不是课程项。

允许的任务来源：

- 搬家与布置
- 宠物照顾
- 商店购买
- 朋友委托
- 校园活动
- 节日准备
- 路线与旅行
- 展览与分享

不允许继续沿用的叙述方式：

- 把迎新或陪伴事件前台写成“课堂导览课”
- 词汇练习
- 语法复习
- 单元测试
- 课堂挑战

这些词保留在这里是为了明确禁用范围，不再作为前台可见表达。

### 2.5 长线系统真相

生活事件/委托完成奖励必须回流到长期系统，而不是停在“一次完成”本身。

固定承接系统：

- `Pet`：喂食、清洁、陪玩、睡觉、外观与用品
- `Shop`：`Pet Shop`、`Clothes Shop`、`Home Decor / General Store`
- `Dress Up`：服装、配饰、包、演出造型
- `Home`：房间角落装饰、功能性小摆件
- `Coins`：常规奖励
- `Parent Bonus`：章节里程碑后的追加激励

## 3. 世界观与总故事

### 3.1 总故事线

故事发生在原创的“星光小镇”。小镇中心的故事钟塔掌管着七段星路灯带，分别连接家的灯、校园的灯、街区的灯、路线的灯和通往远方的灯。

玩家作为新搬来的孩子，先在 `home` 安顿下来，认识自己的第一只宠物，再被 Keeper Ray 邀请帮助小镇恢复失去活力的七段星路。

孩子前台感知到的不是“去学英语”，而是：

1. 先把自己的家收拾好。
2. 先学会照顾宠物。
3. 去认识朋友和常去的地方。
4. 去商店、广场、学校和街区帮忙。
5. 逐步从家门口走到更大的世界。

每修复一段星路，小镇就发生一次可见变化：

- 新区域亮起
- 新商店开放
- 新宠物用品可买
- 新服装或装饰解锁
- 新旅行路线可走
- 旧锚点开始承担新功能

最终，七段星路汇成一张“毕业旅行星图”，玩家要把小学阶段学到的表达能力用在一次旅行分享会里。

## 4. 序章：Letters, Home, My First Pet

### 4.1 序章定位

序章是独立基础层，不属于三下到六下教材主线，但必须先于七篇存在。

序章要解决四件事：

1. 让孩子先把产品理解成“家 + 宠物 + 探索”的游戏，而不是课程。
2. 建立 `A-Z -> 图像 -> 地点` 的最初记忆方式。
3. 把最基础的生活名词、动作词和指示表达先变成熟悉体验。
4. 给后续所有任务提供宠物、家园、商店和回访的动机基础。

### 4.2 序章三段结构

| 段落 | 事件主线 | 语言重点 | 主场景 | 玩法形态 | 结果 |
|---|---|---|---|---|---|
| Letters | 搬家箱里掉出一套字母卡，孩子需要把它们送回正确位置 | A-Z、字母名、少量核心图像词 | `home` 房间、院子、门口锚点 | 指认、找物、简单拖拽 | 建立字母与图像、地点的第一层绑定 |
| Home | 玩家开始整理自己的房间、常用物和出门路线 | 房间词、基础物品词、`this / that`、`go / come / find` | 卧室、客厅、厨房、门口、院子 | 找物、归位、短路线、简单选择 | 建立“家是起点”的稳定体验 |
| My First Pet | 玩家遇见并照顾第一只宠物，完成第一笔生活采购 | 宠物词、食物词、基础动作词、`feed / play / clean / sleep` | 宠物角、厨房、院子、杂货店 / 宠物店入口 | 喂食、清洁、陪玩、购买、回家布置 | 开启宠物系统、商店感知和奖励回流 |

### 4.3 序章生活事件/委托基线

| 序章任务 | 儿童端事件包装 | 地图落点 | 核心玩法 | 结果与解锁 |
|---|---|---|---|---|
| 0-1 | 搬家欢迎箱：找到第一批字母卡和名字牌 | 卧室 / 客厅 | 指认、拖拽 | 建立角色名和字母感知 |
| 0-2 | 苹果架和书包架：把常用物品放回家里 | 厨房 / 书桌角 | 找物、归位 | 建立最早的生活名词组 |
| 0-3 | 房间贴贴纸：认识 bed、bag、book、door 等家居点位 | 卧室 / 门口 | 指认、点击 | 建立 `this / that` 与房间锚点 |
| 0-4 | 第一次出门：从家走到院门，再走到学校门口 | 院子 / `school` 门口 | 短路线、点击移动 | 建立 `go / come / here / there` 感知 |
| 0-5 | 遇见第一只宠物：给它起名并带回宠物角 | 院子 / 宠物角 | 选择、命名、互动 | 解锁宠物常驻 |
| 0-6 | 宠物第一餐：找对食物、食盆和睡垫 | 厨房 / 宠物角 | 喂食、搭配 | 解锁 `feed / eat / drink / sleep` 动作链 |
| 0-7 | 宠物洗香香：完成清洁和陪玩 | 浴室角 / 院子 | 清洁、陪玩 | 解锁心情和清洁状态 |
| 0-8 | 去小店买用品：买第一只碗、小球或小毯子 | `town` 边缘小店入口 | 购买、选择 | 建立 `shop` 感知和 Coins 首次回流 |
| 0-9 | 钟塔欢迎夜：把序章锚点连成第一条小光路 | `home` 门口 / 钟塔远景 | 回顾、连线、点击 | 序章完成，A-Z 全量锚点正式开放 |

### 4.4 序章完成后的系统状态

序章完成后，后续章节默认继承以下状态：

1. 宠物系统已开启，可作为回访动机持续存在。
2. `Coins` 已进入常规循环，孩子知道奖励可以回流到宠物、外观和家园。
3. 小镇地图中的全部 A-Z 锚点位置正式固定并可交互。
4. `home` 成为长期任务起点、奖励投放点和情绪安全区。

## 5. 七篇主线详细设计

### 第一篇：小镇新朋友

`剧情目标`：认识小镇伙伴，完成第一次从家到学校、花园和公共广场的生活跨出。  
`语言目标`：问候、时间、天气、数量、基础交通和最常见生活表达。  
`地图主场景`：`school` 周边、花园、小镇门口、儿童节广场。  
`核心玩法形态`：迎新、出门准备、派对协作、第一次小镇集体活动。

| 教材承接 | 儿童端事件包装 | 地图落点 | 核心玩法 | 章节作用 |
|---|---|---|---|---|
| 三下 U1 `How are you?` | 搬家欢迎日：和新朋友完成第一次见面礼 | 小镇门口 | 对话选择 | 建立主角和伙伴关系 |
| 三下 U2 `What time is it?` | 赶铃声的一天：帮 Mina 调好出门钟和睡前钟 | `home` / `school` 门口 | 时间选择、排序 | 把时间表达变成生活节奏 |
| 三下 U3 `What's the weather like?` | 看天气出门：替自己和宠物挑对当天装备 | 院子 / 花园 | 天气匹配、物品选择 | 让天气词与真实出门决策绑定 |
| 三下 U4 `Do you like the dress?` | 欢迎派对试衣间：为派对选衣服和小配饰 | 衣帽间 / 广场后台 | 换装、喜好选择 | 引入 dress up 动机 |
| 三下 U5 `How many kites are there?` | 花园风筝赛：帮管理员清点风筝和小鸟 | 花园 | 数量清点 | 让数字进入活动现场 |
| 三下 U6 `There is an egg here` | 厨房小点心：帮 Grandma Joy 摆好食材篮 | 厨房 | 找物、分类 | 建立厨房与食物词组 |
| 三下 U7 `Wash your hands` | 宠物晚餐前准备：完成洗手和清洁流程 | `home` / 宠物角 | 动作排序 | 强化日常卫生与动作词 |
| 三下 U8 `Children's Day` | 儿童节小舞台：和彩旗、节目卡一起准备开场 | 活动广场 | 配对、布置 | 第一篇章末集体活动 |
| 三下 U9 `I go there by bus` | 第一次去城里：选好去博物馆或集市的交通方式 | 小镇车站 | 路线选择 | 打开 `transport` 的最早感知 |
| 三下 U10 `Is she a driver?` | 路上认识工作者：记住常见职业和他们在做什么 | 车站 / 街口 | 人物识别 | 为后续 town 生活埋职业线 |

### 第二篇：校园发现日

`剧情目标`：把学校从“去过一次的地方”升级成“能反复生活和帮忙的空间”。  
`语言目标`：学校场所、作息、食物、动物、能力、规则、归属和家庭相关表达。  
`地图主场景`：`school` 核心簇、图书馆、操场、朋友家、`home` 回访点。  
`核心玩法形态`：带路、认路、整理、失物招领、食堂选择、规则理解、拜访朋友。

| 教材承接 | 儿童端事件包装 | 地图落点 | 核心玩法 | 章节作用 |
|---|---|---|---|---|
| 四上 U1 `This is our school` | 陪新同学认路：带他认识教室、操场和图书馆 | 校园入口 / 教学楼 | 场景点击 | 固定学校主场景记忆 |
| 四上 U2 `What time do you get up?` | 校园作息钟：帮老师把一天的铃声排正确 | 教室 / 走廊 | 时间表排序 | 让作息进入校园规则 |
| 四上 U3 `I like dumplings` | 食堂午餐票：为朋友搭配喜欢的午餐 | 食堂 | 点餐选择 | 用食物喜好驱动社交 |
| 四上 U4 `Look at the elephant` | 动物观察角：完成自然课的观察卡 | 校园角 / 花园 | 观察、猜测 | 把动物词放进活动卡 |
| 四上 U5 `Can you play basketball?` | 操场挑战日：给同学分配合适的运动任务 | 操场 | 能力配对 | 强化 `can / can't` 的使用场景 |
| 四上 U6 `What are those?` | 花园远近找物：帮老师分辨近处和远处的植物 | 花园 | 远近分类 | 让 `these / those` 落在可见物体上 |
| 四上 U7 `Whose watch is this?` | 失物招领台：把手表、书包和钥匙送回主人 | 校园服务台 | 线索匹配 | 建立归属关系和人物回忆 |

## 6. 生活事件/委托配置与地图扩展规则

### 6.1 每个事件必须具备的内容真相

后续无论是手写任务、表格配置还是工具生成，每个生活事件或委托都至少要回答以下问题：

| 真相项 | 说明 |
|---|---|
| 章节归属 | 属于序章还是七篇中的哪一篇 |
| 剧情理由 | 角色为什么现在需要做这件事 |
| 儿童端事件名 | 前台如何命名，必须是生活事件 |
| 语言目标 | 内部内容层承接哪些词、句型或语法 |
| 地图主场景 | 任务发生在什么区域、用到哪些锚点 |
| 核心玩法形态 | 选择、拖拽、路线、购买、对话、照顾、布置等 |
| 奖励回流 | 奖励回到 Coins、宠物、服装、家园还是章节主线 |
| 世界变化 | 完成后地图、角色、商店或展台发生什么变化 |
| 回访锚点 | 后续哪些旧锚点会再次调用这个知识点 |
| 家长说明 | 家长层如何解释孩子今天练到了什么 |

### 6.2 地图扩展规则

| 区域 | 可以扩展的故事类型 | 必须遵守的边界 |
|---|---|---|
| `home` | 搬家、房间布置、宠物照顾、早餐、睡前准备 | 维持安全区定位，不改主锚点，不塞成课堂入口 |
| `school` | 带路、认路、作息、社团、失物招领、图书馆、运动和节日现场 | 重点是伙伴和规则，不把学校做成整款产品的脸 |
| `town` | 商店、点餐、借物、巡查、爱好展台、节日市集 | 商店必须属于世界的一部分，不做独立抽象菜单 |
| `transport` | 问路、乘车、送客、路线安排、打包出行 | 负责连接地图扩展，不单独悬空存在 |
| `world` | 旅行展、世界介绍、自然和公共行动、毕业分享 | 作为视野延展区，和前面章节收集内容形成回访闭环 |

### 6.3 回访规则

每个章节至少要承担一种旧知识回访职责：

1. 旧字母锚点在新任务里再次出现。
2. 旧单词在新场景里承担新功能。
3. 旧地点被重新赋予新的剧情用途。
4. 宠物、商店或装扮系统把旧词再次调出。

### 6.4 奖励规则

奖励不能停留在“星片”本身，至少要回流到以下方向之一：

1. 宠物用品
2. 服装或配饰
3. 家园装饰
4. 商店可购买资格
5. 章节展示物
6. 里程碑后的 `Parent Bonus`

## 7. 儿童端文案与任务命名规则

1. 每个任务只给儿童一个明确行动目标，不同时塞入多条教学话术。
2. 标题优先写成事件名，例如“去小店买宠物碗”“帮朋友挂生日卡”“给客人带路”，不要写成知识点名。
3. 不在儿童前台显式写“练习句型”“学习单词”“完成复习”。
4. 三四年级优先图像、物品、路线和动作；五六年级逐步增加计划、调查、问路、记录和展示。
5. 错误反馈只给线索和再试机会，不出现考试式判断。
6. 每个核心语言点至少要在 `主线任务 + 奖励物 / 场景物 + 回访事件` 中出现三次。

## 8. 家长层输出口径

| 阶段 | 家长可见能力描述 |
|---|---|
| 序章 | 孩子已能把字母、常见物品、家和宠物的基础表达与真实互动联系起来。 |
| 第一篇 | 孩子开始能在问候、时间、天气、数量和出行等基础生活场景里使用英语。 |
| 第二篇 | 孩子能在校园和家庭场景中理解场所、喜好、能力、归属和公共规则。 |
| 第三篇 | 孩子能在节日和校园活动中理解位置、课程安排、星期、正在发生的动作和安全标志。 |
| 第四篇 | 孩子能在街区生活中进行点餐、借物、购物、照顾宠物、表达情绪并参与节日筹备。 |
| 第五篇 | 孩子能用英语做计划、排步骤、问路、记录日期和回忆过去经历。 |
| 第六篇 | 孩子能在公共行动中使用过去时、将来时、建议、比较和健康环保表达。 |
| 第七篇 | 孩子能把小学阶段英语能力整合进旅行、展示、演出和毕业分享。 |

## 9. 本稿的使用方式

后续如果继续扩展主线、地图或回访任务，默认按以下顺序判断：

1. 先看它属于 `home / school / town / transport / world` 的哪一段世界路径。
2. 再看它属于 `序章 + 七篇` 中哪一段星路逻辑。
3. 再决定它的儿童端事件包装，而不是先写教材名。
4. 最后才把语言目标挂回配置层和家长层。

如果未来新增内容不能同时回答“它为什么是生活事件”“它落在哪个地图区域”“奖励回流到哪里”“它如何回访旧锚点”，就说明它还不够符合本稿的真相要求。
